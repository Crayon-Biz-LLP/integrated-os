Rhodey (Integrated-OS)

Executive Command System — Vercel Serverless + GitHub Actions + Supabase

Input Channels (7)

  • • Telegram webhook — primary real-time input
  • • Gmail / Outlook email ingestion (GA jobs)
  • • Google Forms journal → Sheets pipeline
  • • Web UI QuickChat + QuickCommand
  • • Scheduled Pulse cron (GitHub Actions)
  • • Sentinel 5-min watcher (external cron-job.org)
  • • WhatsApp / Call recording ingest (auxiliary)

Core Processing (Vercel)

  • • 6-stage webhook: dedup → auth → multimodal → shortcode → clarify → classify
  • • Gemini Flash Lite intent classification
  • • Routing: TASK / NOTE / QUERY / COMPLETION / BRIEF / DELEGATE / PRACTICE / NOISE
  • • interrogate_brain(): 14-source parallel context fetch
  • • Truth Boundary: claim validation + receipt appending

Data Layer

  • • Supabase: tasks, memories (pgvector), graph_nodes/edges, projects, people, messages
  • • Associative retrieval: 7-signal ranking (semantic, PPR, recency, importance, project, specificity)
  • • Knowledge Graph: person / org / project / place / animal — 16 edge types, all HITL-approved
  • • Redis: 1h LLM result cache, 24h embedding cache via Upstash REST API

Safeguards

  • • Truth Boundary — CLAIM_LEXICON + RESERVED_ACTION_PATTERNS regex
  • • Context Registry — 6 strategies with hard/soft/none entity-grounding gates
  • • JSON fail-closed — deterministic safe text on parse failure
  • • Multi-key Gemini failover (3 keys), 3-retry dead-letter queue

Deployment

  • • Vercel serverless (Python FastAPI) — 60s timeout
  • • GitHub Actions — 7 workflows, cron-scheduled, no timeout
  • • External cron-job.org — Sentinel (5min) + Decision Pulse (30min)
  • • Two Vercel projects: backend (root) + frontend (frontend/)
  • • Rewrites scoped per project, never platform-level routes
